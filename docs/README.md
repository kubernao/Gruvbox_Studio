# Gruvbox Studio Documentation

This directory is the working documentation hub for Gruvbox Studio. The goal is to keep project knowledge close to the code and easy to maintain, with clear guides for onboarding, development, testing, and debugging. Each document is scoped to one responsibility so you can quickly find what you need without reading unrelated details.

## Start here

- `architecture.md`: High-level system design and module boundaries.
- `development-workflow.md`: Local setup, daily commands, and packaging flow.
- `testing.md`: Unit, integration, and E2E testing strategy and commands.
- `troubleshooting.md`: Fast fixes for common local development issues.

## Documentation principles

- Keep docs practical: explain current behavior, not aspirational behavior.
- Prefer small updates in the same PR as code changes.
- Use exact command and file names to avoid ambiguity.
- Link to code paths when a behavior depends on implementation details.
